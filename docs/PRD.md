## 1. 배경 및 문제 정의

### 1.1 문제

취준생은 면접을 준비할 때 모의면접을 진행할 수 있는 환경 자체가 없다. 사람과 함께하는 면접 스터디는 일정 조율이 어렵고 피드백 품질도 일정하지 않다. 유료 컨설팅은 비용 부담이 크다.

> **핵심 문제: 사람 의존 없이, 언제 어디서든 모의면접을 할 수 있는 환경이 없다.**

### 1.2 솔루션

실제 면접은 **직무(역량) 인터뷰 + 컬쳐핏 인터뷰** 두 트랙으로 나뉜다. 직무 인터뷰는 내 이력서/포트폴리오를 읽은 면접관이 진행한다. 기존 서비스들은 "나를 모르는 면접관"과의 generic한 연습만 제공한다.

리허설은 JD + 이력서 + 포트폴리오를 학습한 **"나를 아는 AI 스페셜리스트 면접관"** 과의 모의면접 환경을 제공한다.

### 1.3 서비스명 의미

> **리허설 (Rehearsal)**
> 
> - 발음을 흘리면 **"이력서"** 가 들린다
> - RE + HEAR = "다시 듣게 하다" → 면접관이 듣고 싶은 말을 다시 들려주는 연습
> - 한자로 **履說(리설)** = 이력을 말하다

## 2. 목표

### 2.1 제품 목표

- IT 직군 경력직/신입 취준생이 언제 어디서든 실전과 유사한 모의면접을 경험할 수 있게 한다
- JD와 내 이력에 기반한 맥락 있는 질문으로 실질적인 연습 효과를 만든다
- 면접 후 구체적인 피드백으로 개선 방향을 제시한다

### 2.2 MVP 핵심 지표

|지표|설명|
|---|---|
|**면접 완주율**|면접 시작 후 끝까지 완료한 비율|
|**피드백 열람률**|면접 완료 후 피드백 화면 진입 + 실제 열람 시간|
|**7일 내 재방문율**|첫 면접 후 7일 내 다시 돌아와 면접을 재진행한 비율|

## 3. 타겟 유저

### 3.1 Primary

**IT 직군 경력직 이직 준비자**

- 직군: 개발자, PM, 디자이너, AI, 인프라
- 서류 합격 후 면접을 앞둔 시점
- 실전 면접 준비를 위한 맞춤형 모의면접이 필요한 사람

### 3.2 Secondary

**IT 직군 신입 취준생**

- 면접 경험이 부족하고 피드백 받을 곳이 없는 사람

### 3.3 트리거 시점

서류 합격 후 ~ 면접 전

## 4. 범위 (Scope)

### 4.1 MVP In Scope

**[인증]**

- Google 소셜 로그인

**[문서 관리]**

- 이력서/경력기술서, 포트폴리오 업로드 후 계정에 저장
- 언제든 삭제/재업로드 가능 (삭제 시 DB + Storage 완전 삭제)

**[입력]**

- JD: 텍스트 직접 입력 또는 URL 크롤링
- 이력서/경력기술서: 필수 (PDF, DOCX)
- 포트폴리오: 선택 ("포트폴리오 없음" 옵션 제공)
- Git 링크: 선택 (README + 스택 정보만 파싱)

**[면접 설정]**

- 면접 시간: 슬라이더 30분 ~ 90분, 5분 단위
- 면접관 페르소나: 스타트업 실무진 / 대기업 인사팀 / 압박 면접관

**[시뮬레이션]**

- AI 면접관과 텍스트 기반 대화형 직무 인터뷰
- 유저 답변에 따른 꼬리질문
- 힌트 버튼 온디맨드 (질문 의도 + 좋은 답변 구성 요소)
- 중간 이탈 시 이어하기 가능

**[리포트]**

- 답변별 평가: 논리성 / 구체성 / 직무 적합성
- 전체 총평
- 개선 필요 답변 Top 3
- 모범 답변 예시
- 재도전 추천 질문

**[히스토리]**

- 면접 기록 최근 10회 저장
- MVP 전체 무료

### 4.2 MVP Out of Scope (v2 이후)

- 컬쳐핏 인터뷰
- 음성 입력 / STT
- 영상 녹화 및 표정 분석
- Git 코드/커밋 분석 (MVP는 README만)
- 모바일 앱
- BM / 유료 플랜

## 5. 유저 플로우

```
[STEP 1] 로그인
  └─ Google 소셜 로그인

[STEP 2] 문서 업로드 (최초 1회, 이후 재사용)
  └─ 이력서/경력기술서 업로드 (필수)
  └─ 포트폴리오 업로드 (선택 / "없음" 옵션)
  └─ Git 링크 입력 (선택)

[STEP 3] 면접 설정
  └─ JD 입력 (텍스트 or URL)
  └─ 면접 시간 선택 (슬라이더 30~90분)
  └─ 면접관 페르소나 선택

[STEP 4] AI 컨텍스트 분석 (백그라운드)
  └─ JD 핵심 역량 추출
  └─ 이력서 주요 경력/프로젝트 추출
  └─ JD ↔ 이력서 매핑 (강점/갭 식별)
  └─ 질문 세트 생성

[STEP 5] 모의면접 시뮬레이션
  └─ AI 면접관 인트로
  └─ 질문 → 유저 답변 → 꼬리질문 반복
  └─ 힌트 버튼 온디맨드
  └─ 중간 이탈 시 이어하기 가능
  └─ 시간 종료 또는 유저 종료

[STEP 6] 리포트
  └─ 답변별 평가
  └─ 전체 총평 + 개선 필요 답변 Top 3
  └─ 모범 답변 예시
  └─ 재도전 추천 질문
  └─ 다시 연습하기 / 새 면접 시작하기
```

## 6. 기능 명세

### 6.1 문서 관리

|항목|상세|
|---|---|
|지원 형식|PDF, DOCX|
|파일 크기 제한|파일당 10MB 이하|
|저장 방식|계정에 저장, 면접 시 재사용|
|삭제|DB + Storage 완전 삭제 처리|
|Git 파싱 범위|README.md, 언어 비율, 주요 스택 (코드 분석 제외)|

### 6.2 면접관 페르소나

|페르소나|말투|질문 스타일|
|---|---|---|
|**스타트업 실무진**|캐주얼하고 직접적|실제 업무 능력, 빠른 적응력 중심|
|**대기업 인사팀**|정중하고 격식체|역량 모델 기반, 조직 적합성 중심|
|**압박 면접관**|날카롭고 도전적|모순 지적, 재질문, 침묵 활용|

### 6.3 힌트 기능

- 노출 방식: "힌트 보기" 버튼 클릭 시 표시 (온디맨드)
- 힌트 내용:
    - 질문 의도 ("이 질문은 OO 역량을 보려는 것입니다")
    - 좋은 답변 구성 요소 ("STAR 구조, 수치 포함 권장")

### 6.4 AI 컨텍스트 분석

3단계 프롬프트 구조:

1. **분석 프롬프트**: JD + 이력서 → 핵심 역량 매핑 + 질문 세트 생성
2. **면접관 프롬프트**: 페르소나별 시스템 프롬프트 + 대화 컨텍스트 유지
3. **평가 프롬프트**: 전체 대화 이력 → 답변별 평가 + 리포트 생성

### 6.5 리포트

**답변별 평가 항목:**

|평가 축|설명|
|---|---|
|논리성|두괄식 구성, STAR 구조 여부|
|구체성|수치/사례 활용 여부|
|직무 적합성|JD 요구 역량과의 연관성|

**리포트 구성:**

- 전체 총평
- 개선 필요 답변 Top 3 + 개선 방향
- 모범 답변 예시 (열람 가능)
- 재도전 추천 질문 ("이 질문 다시 연습할래요?" 버튼)

### 6.6 히스토리

- 최근 10회 저장
- 10회 초과 시 가장 오래된 기록부터 삭제 확인 후 처리
- 이어하기: 중간 이탈한 면접 재진입 가능

## 7. 기술 스택

|영역|기술|
|---|---|
|**IDE**|Google Antigravity|
|**Framework**|Next.js|
|**UI**|shadcn/ui|
|**AI 모델**|Gemini 3 Pro|
|**Database**|Supabase|
|**Auth**|Supabase Auth (Google OAuth)|
|**Storage**|Supabase Storage (무료 티어, 1GB)|
|**배포**|Vercel|

### 7.1 기술적 리스크 및 대응

|리스크|내용|대응 방안|
|---|---|---|
|문서 파싱 품질|스캔본 PDF, 이미지 이력서 추출 실패|텍스트 직접 붙여넣기 옵션 병행 제공|
|컨텍스트 길이|문서가 많을 경우 토큰 초과|청크 분리 후 요약본만 컨텍스트에 포함|
|질문 중복|AI가 유사한 질문 반복 생성|질문 세트 사전 생성 후 중복 필터링|
|응답 레이턴시|AI 응답 지연으로 면접 몰입감 저하|스트리밍 응답, 로딩 상태 UX 처리|
|Storage 용량|유저 증가 시 1GB 초과|Cloudflare R2 마이그레이션 검토|
